环境准备流程源码

SparkSubmit

  1. 当调用 spark-submit 后，它会依次调用 spark-submit2 和 spark-class2
  2. 在 spark-class2 最后将调用 %SPARK_CMD%，在源文件中添加代码 echo %SPARK_CMD%，控制台将输出程序最终执行代码
  3. 在 cmd 中运行提交程序，可以看出 spark-submit 最终会调用执行 org.apache.spark.deploy.SparkSubmit 类的 main 方法
想 Yarn 提交应用
  1. SparkSubmit 的 main 方法将调用 doSubmit 方法
  2. doSubmit 调用 parseArguments(args) 解析传入参数，该方法通过创建 SparkSubmitArguments 类来解析
    1. 解析传入参数 –master 和 –class 等
    2. 设置 action 默认为 SUBMIT action = Option(action).getOrElse(SUBMIT)
  3. appArgs.action match 将匹配到 SUBMIT 调用 submit(appArgs, uninitLog) 方法
  1. submit 方法将调用 runMain
  2. runMain 将根据 childMainClass 通过反射创建出具体对象 app 并调用 start 方法
  3. childMainClass 的值是通过 prepareSubmitEnvironment(args) 方法获取，如果是 yarn 集群的话，将返回 “org.apache.spark.deploy.yarn.YarnClusterApplication”
  1. YarnClusterApplication 的 start 方法将会创建一个 yarnClient
  2. yarnClient 通过 start 与 Yarn Cluster 的 ResourceManager 连接
  3. yarnClient 又会通过 submitApplication(appContext) 向 ResourceManager 发送一些列指令 commands
  4. 指令 commands 通过 “/bin/java” + “org.apache.spark.deploy.yarn.ApplicationMaster” 拼接而成，说明 ResourceManager 将会在一个 NodeManager 中启动 ApplicationMaster 程序
ApplicationMaster 启动 Driver 线程
  1. ApplicationMaster 会创建 ApplicationMaster 对象，该对象中有一个属性为 client 将创建用于连接 Yarn Cluster 的 ManagerResource 的客户端 YarnRMClient
  2. ApplicationMaster 对象将被调用 run 方法，间接调用了 runDriver 方法
  3. runDriver 方法将调用 startUserApplication 返回一个线程，该线程名为 Driver 中通过 mainMethod.invoke(null, userArgs.toArray) 反射调用了传入 –class 类的 main 方法
ApplicationMaster 启动 Executor 进程
  1. val sc = ThreadUtils.awaitResult(sparkContextPromise.future, Duration(totalWaitTime, TimeUnit.MILLISECONDS))将会阻塞等待 Driver 返回 SparkContext 对象
  2. registerAM(host, port, userConf, sc.ui.map(_.webUrl), appAttemptId) 会将 ApplicationMaster 注册到 Yarn Cluster 的 ResourceManage 中
  3. createAllocator 可以得到 ResourceManage 返回的可用资源列表
  4. 当可用资源列表数大于 0 时，handleAllocatedContainers 将会对可用资源进行处理
  5. 可用发现，ApplicationMaster 会使用线程池 launcherPool 执行 ExecutorRunable 的 run 方法
  6. run 方法中就是创建了一个 NodeManageClient，并通过 startContainer 创建容器
  7. startContainer 会向 NodeManageClient 发送由 prepareCommand 生成的指令 commands
  8. commands 为 “/bin/java” + “org.apache.spark.executor.YarnCoarseGrainedExecutorBackend”
